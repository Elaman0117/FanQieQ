# 手动工作流操作指南

> **适用场景**：AI不支持agent功能，需要手动复制粘贴上下文。
> **核心思路**：用户充当"人肉agent"，按"检索→生成→更新"三步循环。
> **每章工作量**：约2-3轮对话 + 手动粘贴更新。

---

## 一、总览：每章的对话轮次

```
第1章（首次启动）：3轮
  第1轮：加载系统提示词 + 请求检索清单
  第2轮：喂词条 + 生成章节
  第3轮：请求世界书更新

第2章及以后（常规）：2轮
  第1轮：喂上下文 + 生成章节
  第2轮：请求世界书更新

每5-10章：额外1轮
  重新加载01系统提示词（防止AI遗忘）
```

## 四、手动检索指南

### 4.1 如何判断需要哪些世界书词条？

**判断依据**：根据章节任务指令中的6个维度

| 维度 | 看任务指令的哪部分 | 需要检索的词条 |
|------|-----------------|--------------|
| 出场角色 | "出场角色"栏 | CHAR_/VILAIN_ 对应角色 |
| 场景 | "场景"栏 | GEO_BUILD_ 对应场景 |
| APP使用 | "APP提示/兑换/升级"栏 | APP_RULE_危机分类/等级/格式 + POINT_PRICE |
| 善意值 | "APP兑换"栏 | POINT_PRICE + STAT_林舟_已兑换能力 |
| 伏笔 | "本章伏笔"栏 | PLOT_FX_ 对应伏笔 |
| 上一章衔接 | 自动 | CHAP_[上一章]_事件 |

## 五、长期维护

### 5.1 对话过长怎么办？

**症状**：AI回复变慢、遗忘早期设定、回复质量下降。

**处理**：
1. 每写完5-10章，开新对话
2. 新对话第一轮重新粘贴01系统提示词
3. 喂最近3章正文 + 当前章节任务指令 + 相关词条

### 5.2 开新对话的启动模板

```
[粘贴01系统提示词全文]

---

我正在写一本番茄小说，已经写到第X章。以下是当前状态：

## 最近3章正文

### 第X-2章
[粘贴正文]

### 第X-1章
[粘贴正文]

### 第X章（上一章）
[粘贴正文]

## 第X+1章任务指令
[填写任务指令]

## 相关世界书词条
[粘贴相关词条]

---

请生成第X+1章正文。
```

### 5.3 世界书词条累积过多怎么办？

**症状**：某些词条文件越来越大（如CHAP_事件累积到50个）。

**处理**：
1. 每50章归档一次旧章节事件
2. 把Ch1-50的CHAP_词条移到 `世界书/章节事件归档/` 文件夹
3. 检索时只检索最近10章的CHAP_词条
4. 远距离伏笔通过PLOT_FX_词条检索，不需要回查原文

### 5.4 设定冲突怎么办？

**症状**：AI生成的章节与已写章节或世界书冲突。

**处理**：
1. 立即停止写作
2. 在对话中告知AI冲突详情
3. 让AI建议修订方案
4. 修订章节正文
5. 更新对应世界书词条

## 七、常见问题

### Q1：01系统提示词太大，每次都要粘贴吗？

**A**：
- 同一对话内，只需第1轮粘贴一次
- 开新对话时，必须重新粘贴
- 如果对话超过10章，建议开新对话重新粘贴

### Q2：世界书词条越来越多，都要粘贴吗？

**A**：
- 每章只粘贴"相关词条"，不是全部
- 按"检索清单"判断需要哪些
- 通常每章粘贴5-10个词条

### Q3：AI输出的更新内容太长，怎么办？

**A**：
- 让AI分批输出（先输出CHAP_事件词条，再输出其他）
- 或者让AI只输出"变化部分"（如POINT_PRICE只输出新增的行）

### Q4：如何确认AI没有遗忘设定？

**A**：
- 每10章做一次自检（让AI对照01文件第十一层+世界书词条）
- 自检报告发现问题时，重新喂相关词条

### Q5：章节任务指令不会填怎么办？

**A**：
- 参考01文件第十三层"早期细致大纲"对应章节
- 复制大纲内容到任务指令模板中
- 大纲已包含每章的事件/钩子/伏笔

### Q6：AI生成质量下降怎么办？

**A**：
- 检查是否遗忘01系统提示词 → 重新粘贴
- 检查是否遗漏关键世界书词条 → 补充粘贴
- 检查章节任务指令是否清晰 → 重新填写
- 暂停写作，让AI重读01文件中的写作规则

## 八、效率优化建议

### 8.1 批量生成章节任务指令

- 一次填写5-10章的任务指令（参考01文件第十三层大纲）
- 保存为单独文件
- 每章开写时直接复制对应部分

### 8.2 预先准备词条包

- 把常用词条（CHAR_HERO_林舟、APP_RULE_*、GEO_BUILD_校医室）合并成一个"常用词条包"
- 每章开写时先粘贴常用包，再粘贴特殊词条

### 8.3 使用文本替换工具

- 用文本扩展工具（如AutoHotkey、TextExpander）存储常用词条
- 输入快捷键自动粘贴词条内容

### 8.4 章节正文模板

- 准备一个章节正文模板（含开头/主体/结尾结构提示）
- 每章开写时先粘贴模板，再让AI填充

## 九、总结

手动工作流的核心是"用户充当agent"，每章2-3轮对话：

1. **第1轮**：喂上下文+生成章节
2. **第2轮**：请求世界书更新
3. **手动操作**：保存正文+更新词条文件

**关键习惯**：
- 每章写完立即更新世界书（不要累积）
- 每5-10章开新对话重新加载01
- 定期自检设定一致性

**效率提升**：
- 批量准备任务指令
- 预先准备常用词条包
- 用文本替换工具

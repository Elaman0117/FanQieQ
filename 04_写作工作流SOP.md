# 写作工作流SOP（标准操作流程）

> **使用方式**：本文件定义从"开写前准备"到"章节生成"到"写完后维护"的完整工作流。
> **目标**：让每章写作都有标准流程可循，避免设定漂移、伏笔遗漏、人设崩塌。

## 三、阶段1：每章开写前

### 3.1 Step 1.1：用户填写章节任务指令

**操作**：用户复制`03_章节任务指令模板.md`中的"标准章节任务指令模板"，填写后发送给AI。

**填写要点**：
1. 章节基础信息：章节号、标题、所属事件弧、定位
2. 本章核心任务：1-2句话概括
3. 本章必写元素：勾选APP提示/兑换/升级、出场角色/女主/反派、场景、对话、细节、钩子
4. 本章大纲细纲：开头-主体-结尾三段
5. 本章伏笔处理：埋设/推进/回收
6. 本章情感节奏：读者心理路径+情绪曲线
7. 本章特殊要求（可选）

**示例**：见`03_章节任务指令模板.md`的"示例1：第7章"。

### 3.2 Step 1.2：AI检索世界书

**操作**：AI收到章节任务指令后，按`02_世界书框架.md`的"5.1每章开写前的检索流程"检索。

**检索顺序**：

**第1步：必读词条**
```
- CHAR_HEROINE_[本章出场女主]
- VILAIN_[本章出场反派]
- APP_RULE_危机等级
- APP_RULE_提示格式
- POINT_PRICE_前50章兑换表
- GEO_BUILD_[本章场景]
- CHAP_[上一章]_事件  # 延续钩子
- STAT_林舟_已兑换能力
- STAT_5女主感情进度总表
```

**第2步：相关词条**
```
- ARC_[当前事件弧]
- PLOT_[本章涉及伏笔]
- STAT_[本章出场角色]_当前状态
- TEMP_[当前事件弧]_专属设定
```

**第3步：特殊词条**
```
- 本章是升华点 → 检索升华点节奏表
- 本章是修罗场 → 检索5女主关系表
- 本章是反派倒戈 → 检索反派行为底层逻辑表
- 本章是APP升级 → 检索APP升级路径表
```

### 3.3 Step 1.3：AI确认设定

**操作**：AI检索完后，向用户报告设定确认。

**确认模板**：
```
收到第X章任务指令。

## 检索世界书
我已检索以下词条：
- [列出检索的词条]

## 确认本章设定
- 善意值余额：XXX（Ch[X-1]后）
- 本章兑换后余额：XXX
- 出场角色状态：[简述]
- 女主感情进度：[简述]
- 伏笔状态：[简述]

如设定无误，我将开始生成章节。
```

## 四、阶段2：章节生成

### 4.1 Step 2.1：AI生成章节正文

**操作**：AI按章节任务指令+世界书检索结果，生成3000字正文。

**章节结构**（教程11.4）：
```
【第X章 标题】

（开头200-300字）
- 延续上一章钩子
- 快速进入本章场景
- 引出本章核心冲突

（主体2400-2500字）
- 本章主要事件（1500-1800字）
- 至少1个冲突/爽点（500-800字）
- 推动至少1条线（主线/感情线/事业线）
- 至少1个APP提示（符合格式）
- 至少1个对话场景
- 至少1个细节描写

（结尾200-300字）
- 解决本章部分问题
- 留新钩子引向下一章
- 可选：升华金句
```

### 4.2 Step 2.2：AI自检

**操作**：AI生成章节后，按以下清单自检。

**自检清单**：
- [ ] 字数是否3000字？（±100字可接受）
- [ ] 开头是否延续上章钩子？
- [ ] 主体是否有事件+冲突？
- [ ] 结尾是否留新钩子？
- [ ] 是否无多余铺垫？
- [ ] 至少1个对话场景？
- [ ] 至少1个细节描写？
- [ ] APP提示是否符合格式？（常规/红色预警/解除反馈三种）
- [ ] 主角是否符合"嘴贱心软"人设？
- [ ] 主角是否遵循"用最大恶意揣测危机，用最大善意对待病人"？
- [ ] 反派是否遵循其行为底层逻辑？
- [ ] 女主感情进度是否符合预设？（不能超前）
- [ ] 善意值数字是否与价格表一致？
- [ ] 场景描写是否与地理词条一致？
- [ ] 伏笔处理是否符合本章预设？

**如自检不通过**：AI立即修订，重新生成。

## 五、阶段3：写完后维护

### 5.1 Step 3.1：AI更新世界书词条

**操作**：AI写完一章后，按`02_世界书框架.md`的"六、AI更新规则（自主判断模式）"执行。

**核心原则**：AI对每条潜在更新拥有完全的自主判断权，可根据本章实际内容决定是否更新/新建/跳过。只有当本章内容确实产生了"会影响后续章节写作的信息"时，才需要更新/新建。

**候选更新清单**（AI按需勾选）：
1. 新建`CHAP_[本章]_事件`：通常执行（章节摘要）
2. 更新`STAT_林舟_已兑换能力`：如有新兑换
3. 更新`POINT_PRICE_前50章兑换表`：如有善意值变动
4. 更新`STAT_[出场角色]_当前状态`：如有显著变化
5. 更新`STAT_5女主感情进度总表`：如有感情进展
6. 新建`CHAR_/VILAIN_`词条：如新角色会再出场
7. 新建`GEO_BUILD_`词条：如新场景会再出现
8. 更新`PLOT_FX_`词条：如有伏笔推进/回收
9. 新建`PLOT_FX_`词条：如发现新长线伏笔
10. 更新APP功能解锁状态：如有APP升级
11. 更新`ARC_XX_`状态：如事件弧结束
12. 新建`TEMP_`词条：如需要临时规则

**AI决策流程**：扫描本章内容 → 对照候选清单判断 → 执行更新/跳过/新建 → 生成更新报告

详细规则见`02_世界书框架.md`第6节。

### 5.2 Step 3.2：AI更新善意值对账表

**操作**：每章写完后，AI更新`POINT_PRICE_前50章兑换表`。

**更新内容**：
- 兑换记录：章节+能力+消耗善意值+累计善意值+用途
- 获取记录：章节+危机解除+获得善意值+累计
- 余额对账：当前累计获得-当前累计消耗=当前余额

**示例（Ch7后）**：
```
## 兑换记录
| 章节 | 兑换能力 | 消耗善意值 | 累计消耗 | 用途 |
|-----|---------|----------|---------|------|
| Ch2 | 初阶影像诊断 | 80 | 80 | 扫描李明脑瘤 |
| Ch5 | 心理疏导术Lv1 | 150 | 230 | 救张老师 |
| Ch6 | 博识眼 | 200 | 430 | 扫描苏晚书包 |
| Ch7 | 溯源眼Lv1 | 300 | 730 | 看债务链条 |

## 获取记录
| 章节 | 危机解除 | 获得善意值 | 累计获得 |
|-----|---------|----------|---------|
| Ch3 | 李明脑瘤 | 120 | 120 |
| Ch5 | 张老师抑郁 | 150 | 270 |
| Ch7 | 苏晚债务 | 300 | 570 |

## 余额对账
- 累计获得：570
- 累计消耗：730
- 余额：-160 ❌（不平衡！）

问题：Ch7兑换溯源眼Lv1花300，但累计获得只有570，余额不足。
解决：需要补充Ch7前的危机解除善意值，或调整兑换价格。
```

**重要**：如余额对账不平衡，AI必须立即报告用户，调整设定后再继续。

### 5.3 Step 3.3：AI更新女主感情进度表

**操作**：每章写完后，AI更新`STAT_5女主感情进度总表`。

**更新内容**：
- 各女主在本章的状态（出场/好感启动/暧昧/确认/修罗场）

### 5.4 Step 3.4：AI更新伏笔进度表

**更新内容**：

### 5.5 Step 3.5：AI生成章节摘要

**摘要内容**：

## 六、阶段4：定期自检（每10章一次）

### 6.1 每10章自检流程

**触发**：每写完10章（Ch10、Ch20、Ch30...）后，AI执行一次全面自检。

**自检内容**（15项）：

| 6. 主角底线 | 是否违背3条不能做+3条必须做 | 同上 |
| 7. 场景一致性 | 校医室布局、大学地理、城市命名 | GEO_BUILD_* |
| 8. 穿越设定一致性 | 是否出现"文抄公"、平行世界差异 | 01文件11.5 |

| 10. 女主标签一致性 | 各女主核心标签是否被强化 | CHAR_HEROINE_* |
| 11. APP升级一致性 | APP等级、功能是否按路径解锁 | 01文件12.2 |

## 七、特殊场景处理流程

### 7.1 卡文处理

**症状**：AI生成章节时卡住，无法继续。

**处理流程**：
1. AI报告卡文原因（剧情卡文/细节卡文/状态卡文）
2. 用户参考教程11.2"卡文应对策略"：
   - 剧情卡文：跳过这段，先写后面的
   - 细节卡文：用最简单的话把事件写完
   - 状态卡文：降低当日目标
3. AI根据用户指导继续生成

### 7.2 设定冲突处理

**症状**：AI发现设定冲突（如善意值余额不足、女主进度超前）。

**处理流程**：
1. AI立即停止生成
2. AI报告冲突详情
3. 用户决策：
   - 修改设定（更新世界书）
   - 修改本章大纲（调整任务指令）
   - 修订已写章节（回查并修正）
4. AI按用户决策执行

### 7.3 伏笔遗漏处理

**症状**：自检时发现某伏笔未按计划埋设/推进/回收。

**处理流程**：
1. AI报告遗漏详情
2. 用户决策：
   - 在后续章节补埋/补推进
   - 修订已写章节补入
   - 调整伏笔回收时间
3. AI按用户决策执行

### 7.4 人设崩塌处理

**症状**：自检时发现主角/女主/反派行为违背行为底层逻辑。

**处理流程**：
1. AI立即报告崩塌详情（哪章哪段违背了哪个逻辑）
2. AI建议修订方案
3. 用户确认后，AI修订该章节
4. AI更新世界书相关词条

## 九、效率优化建议

### 9.1 批量写作

- 日常：一次写2章（6000字，符合日更要求）
- 状态好时：一次写3-4章，存稿
- 状态差时：只写1章，降低目标

### 9.2 章节任务指令复用

- 同一事件弧的章节，任务指令可复用模板
- 修改"本章核心任务"和"大纲细纲"即可

### 9.3 世界书检索优化

- AI检索时优先检索最近5章的`CHAP_`词条（保持连贯）
- 远距离伏笔通过`PLOT_F`词条检索，不需要回查原文

### 9.4 自检自动化

- AI每章写完后自动自检（15项中的核心5项）
- 每10章执行完整15项自检
- 自检报告自动存档

## 十、快速启动指南

### 10.1 第一次使用（Ch1开写前）

1. **阅读4个文件**：
   - `01_综合系统提示词.md`（通读3遍）
   - `02_世界书框架.md`（通读2遍）
   - `03_章节任务指令模板.md`（通读1遍）
   - `04_写作工作流SOP.md`（本文件，通读1遍）

2. **创建世界书初始词条**：
   - 按`02_世界书框架.md`的"八、世界书初始词条清单"
   - 创建25个永久词条（P0级）
   - 动态词条和临时词条初始为空

3. **填写Ch1章节任务指令**：
   - 复制`03_章节任务指令模板.md`的标准模板
   - 参考`01_综合系统提示词.md`的第十三层"早期细致大纲"Ch1部分
   - 填写后发送给AI

4. **AI生成Ch1**：
   - AI检索世界书
   - AI生成3000字
   - AI自检
   - AI更新世界书

5. **用户审核Ch1**：
   - 检查是否符合预期
   - 如需修订，按`03_章节任务指令模板.md`的"修订指令模板"

### 10.2 日常写作（Ch2及以后）

1. **填写章节任务指令**（5分钟）
2. **AI检索+生成+更新**（10-15分钟）
3. **用户审核**（5-10分钟）
4. **如需修订，发送修订指令**
5. **进入下一章**

### 10.3 每10章自检

1. **触发自检**（Ch10、Ch20、Ch30...）
2. **AI执行15项自检**
3. **AI生成自检报告**
4. **用户审核报告**
5. **如有问题，按"特殊场景处理流程"修复**

## 十一、应急处理

### 11.1 AI生成质量下降

**症状**：章节质量明显下降（人设崩塌、节奏拖沓、APP格式错误）。

**处理**：
1. 暂停写作
2. 重新加载`01_综合系统提示词.md`
3. 让AI重新检索世界书
4. 从上一章重新开始

### 11.2 世界书词条丢失

**症状**：AI检索不到某个词条。

**处理**：
1. 检查文件是否存在
2. 如不存在，按模板重新创建
3. 从已写章节中恢复信息

### 11.3 设定矛盾累积

**症状**：多次自检发现矛盾，无法修复。

**处理**：
1. 暂停写作
2. 全面回查所有世界书词条
3. 制定修复方案
4. 修订受影响章节
5. 更新世界书

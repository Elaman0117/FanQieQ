# P0 设定（写第1章前必须固定）

## P0-1：校园危机预警APP核心机制

### 教程依据
- 教程2.2要点2：金手指必须**明确、可量化、有成长性**
- 教程3.4强设定第1层：金手指设定要明确规则、限制、成长路径
- 教程4.2开篇禁忌：不能第一时间抛出大量设定，要在解决危机中带出

### v1/v2一致性核对
- v1第七节简介：【校园危机预警】APP + 倒计时格式【高三一班李明：脑部占位性病变概率87%，可干预窗口23天】
- v2卷一Ch1：APP自动下载、删不掉、第一次提示
- v2 P0清单①：APP命名、危机分类、等级规则、倒计时格式、善意值命名

### 详细设定

#### 1.1 APP基础信息

| 子项 | 设定值 | 冲突预防说明 |
|-----|-------|------------|
| APP中文名 | **校园危机预警** | 全书统一，不出现"校园守护""危机雷达"等其他叫法 |
| APP英文名 | **C-CWS**（Campus Crisis Warning System，仅在第29章老主任解释时出现一次） | 避免英文反复出现破坏中文阅读节奏 |
| APP图标 | 红底白色十字+倒计时符号 | 后期如需描写图标，统一用此描述 |
| APP出现方式 | **自动下载、无法卸载** | 不写"觉醒获得"或"穿越自带"，避免与原主设定冲突 |
| APP绑定规则 | 绑定当前手机，手机更换自动迁移 | 避免后期"换手机APP没了"的逻辑漏洞 |
| APP是否需要登录 | 不需要，打开即用 | 避免账号密码情节 |

#### 1.2 APP显示的危机类型分类（共7类，固定）

| 危机类型 | 代号 | 典型场景 | 出现章节 |
|---------|------|---------|---------|
| 疾病危机 | D（Disease） | 脑瘤、抑郁、自杀、兴奋剂 | Ch1李明脑瘤 |
| 心理危机 | P（Psychology） | 抑郁、焦虑、自残倾向 | Ch4张老师 |
| 财务危机 | F（Finance） | 债务、诈骗、催收 | Ch6苏晚 |
| 人际危机 | R（Relationship） | 霸凌、家庭矛盾、情感纠葛 | Ch14林小满 |
| 安全危机 | S（Safety） | 车祸、袭击、绑架 | Ch21沈清 |
| 学业危机 | A（Academic） | 学术造假、考试舞弊、退学 | Ch19周浩 |
| 家庭危机 | H（Home） | 逼婚、家暴、父母问题 | Ch16林小满家 |

**冲突预防**：
- 全书所有危机必须归入这7类，不出现"运气危机""命运危机"等模糊分类
- 每次APP提示必须带代号字母，如【D-李明：脑瘤...】
- 一个危机可同时归多类（如苏晚债务是F+H），但主类型只能选1个

#### 1.3 危机等级命名规则（5级，固定）

| 等级 | 命名 | 颜色 | 干预窗口 | 出现频率 |
|-----|------|-----|---------|---------|
| Lv1 | **低危** | 灰色 | 30天以上 | 高频，背景板 |
| Lv2 | **中危** | 黄色 | 7-30天 | 中频，每3-5章一个 |
| Lv3 | **高危** | 橙色 | 1-7天 | 中频，每个事件弧1-2个 |
| Lv4 | **致命** | 红色 | 24小时内 | 低频，卷一不超过5个 |
| Lv5 | **临界** | 黑色闪烁 | 1小时内 | 极低频，卷一仅2次（Ch5张老师、Ch21沈清） |

**冲突预防**：
- 不使用"1级/2级""S级/A级""★数量"等其他命名
- 等级与颜色绑定，全书不变
- 危机升级路径只能从低到高（如Lv2→Lv3），不能跨级跳（Lv2直接变Lv4）
- 危机解除后等级归零，不出现"等级倒退"

#### 1.4 APP提示格式（3种，固定）

**格式A：常规提示**（最常用）
```
【危机类型代号-姓名：危机描述，概率/风险等级，可干预窗口XX天/小时】
```
示例：【D-李明：脑部占位性病变概率87%，可干预窗口23天】

**格式B：红色预警**（仅Lv4-Lv5）
```
【⚠️红色预警-姓名：危机类型，等级LvX，位置XX，倒计时XX】
```
示例：【⚠️红色预警-张老师：自杀风险，等级Lv5，位置教学楼B栋天台，倒计时47分钟】

**格式C：危机解除反馈**（每次解决后）
```
【危机解除-姓名：进度+XX%，获得善意值XXX】
【APP升级提示：善意值突破XXXX，解锁新功能：XXX】
```
示例：【危机解除-李明：进度+50%，获得善意值120】

**冲突预防**：
- 三种格式全书不混用，常规提示不带⚠️，红色预警必带
- 姓名必须用真名，不用"高三某学生"等模糊表述
- 概率用百分比（0-100%），不用"很高""较低"等模糊词
- 时间窗口用"天"或"小时"或"分钟"，不用"不久""很快"

#### 1.5 APP主动推送规则

| 规则 | 设定 |
|-----|------|
| 主动推送触发条件 | 仅Lv4致命、Lv5临界危机主动推送 |
| Lv1-Lv3 | 需主角主动打开APP查看 |
| 推送时间 | 24小时随时推送，包括深夜 |
| 推送方式 | 手机震动+红色弹窗 |
| 静默模式 | 不存在静默模式（避免主角"没看到"的漏洞） |
| 同时推送数量 | 最多3个，超过会按等级排序 |

#### 1.6 APP的限制（必须设定，避免万能化）

| 限制项 | 设定 | 教程依据 |
|-------|------|---------|
| 距离限制 | 仅显示南江大学校内+周边1公里（卷一） | 教程2.2"金手指要有规则" |
| 人数限制 | 同时显示危机上限20个（避免信息过载） | 同上 |
| 时间限制 | 最远只能预判7天（卷一），卷二解锁30天 | 教程2.2"可量化" |
| 干预限制 | APP只显示危机，不提供解决方案（方案靠善意值兑换） | 避免主角无脑 |
| 代价限制 | 每次使用【危机溯源】消耗50善意值（避免滥用） | 教程3.4"金手指要有规则" |
| 失效场景 | 主角被警方控制时APP暂时失灵（Ch33卷一低谷） | 教程8.3"反派升级" |
| 禁忌 | 主角不能干预自己的危机（避免悖论） | 教程3.4伏笔 |

## P0-2：善意值兑换系统

### 教程依据
- 教程2.2要点2：金手指可量化、有成长性，升级方式清晰
- 教程3.3事业线：能力升级路线Lv1→Lv2→...→LvMax，每个等级有明确能力范围
- 教程3.4强设定第1层：金手指明确规则、限制、成长路径

### v1/v2一致性核对
- v1第七节简介：兑换医术、技能、物品，解锁女主线
- v2 Ch2：花80善意值兑换【初阶影像诊断】
- v2 Ch5：花150善意值兑换【心理疏导术Lv1】
- v2 Ch6：花200善意值兑换【博识眼】
- v2 Ch7：升级【溯源眼】
- v2 P0清单②：善意值与能力对应表、能力分类、等级命名、是否可叠加

### 详细设定

#### 2.1 善意值基础信息

| 子项 | 设定值 | 冲突预防 |
|-----|-------|---------|
| 善意值命名 | **善意值**（全书统一，不出现"功德值""救世点""危机积分"） | 避免与其他小说设定混淆 |
| 善意值单位 | 无单位，纯数字 | 不写"善意点""善意币" |
| 善意值获取方式 | 解决危机后APP自动发放 | 不出现"任务奖励""签到获得" |
| 善意值存储上限 | 卷一10000，卷二50000，卷三无上限 | 避免早期通货膨胀 |
| 善意值是否可交易 | 不可交易、不可转让 | 避免后期"贩卖善意值"漏洞 |

#### 2.2 善意值获取规则（按危机等级）

| 危机等级 | 解除获得善意值 | 备注 |
|---------|-------------|-----|
| Lv1低危 | 10-30 | 小问题，如感冒提醒 |
| Lv2中危 | 50-100 | 中等问题，如轻度抑郁 |
| Lv3高危 | 100-200 | 如苏晚债务 |
| Lv4致命 | 200-500 | 如李明脑瘤 |
| Lv5临界 | 500-1000 | 如张老师自杀、沈清车祸 |

**额外加成规则**：
- 危机涉及多人时，善意值×人数
- 危机背后有反派操纵时，善意值×1.5（卷一Ch10后启用）
- 主角主动预判危机（非APP提示）解决时，善意值×2

**冲突预防**：
- 善意值数字全书递增，不能Ch3给120、Ch20给80（除非有剧情原因）
- 每次危机解除必须明确写出善意值数字，不能模糊"获得不少善意值"

#### 2.3 能力分类体系（5大类，固定）

| 类别 | 命名规则 | 包含能力（前50章） |
|-----|---------|-----------------|
| **医术类** | 【XX诊断LvX】【XX治疗LvX】 | 初阶影像诊断、初级化验术、中级临床判断 |
| **心理类** | 【XX疏导LvX】【XX洞察LvX】 | 心理疏导术Lv1-Lv3、中级心理洞察 |

| **物品类** | 兑换实物（药品、器材、文档） | 卷一仅兑换1次（特殊药品） |

**冲突预防**：
- 能力命名必须带等级标识（Lv1/Lv2/初阶/中阶），不出现"超级医术""神级感知"

#### 3.4 行为底层逻辑（教程6.4方法1：通篇遵循不可违背）

**行为底层逻辑：用最大恶意揣测危机，用最大善意对待病人**

**含义**：
- 对危机本身：永远假设最坏情况，APP显示87%概率脑瘤，他要按100%处理
- 对反派：永远假设对方有后手，赵天宇威胁时他要预判对方下杀手
- 对病人/被救者：永远给予最大善意，无论对方态度多差都救
- 对自己：永远把自己放在最后，可以冒犯所有人，但绝不放弃任何一个能救的人

**应用示例**：

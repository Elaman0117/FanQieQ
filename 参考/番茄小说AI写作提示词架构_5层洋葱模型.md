# 番茄小说AI写作提示词架构（5层洋葱模型）

> **目的**：建立一套可复用的AI写作提示词架构，让每一章的写作都能稳定输出符合设定和教程的内容。
> **适用**：Claude/GPT/Gemini 等长上下文模型，特别适合番茄小说日更6000字（2章/日）场景。
> **核心问题**：教程80KB + 设定集64KB + 大纲52KB + 已写章节，每章都全塞进上下文不现实，必须有架构。

---

## 第一部分：评估你的3层方案

### 你的方案回顾
1. **小说已经存在的部分**：已写章节
2. **综合提示词**：教程写作原则 + 整体大纲 + 早期细致大纲 + 基础设定
3. **世界书/设定集相关部分**：与当前章节相关的设定

### 优点
- ✅ 三层逻辑清晰：已有 / 原则 / 细节
- ✅ 区分了"会变的"（已写章节）和"不变的"（原则、设定）
- ✅ 意识到"按需调用设定"，避免每次塞全本

### 三个核心问题

**问题1：已有内容如果不压缩，30章后超上下文**
- 每章3000字 ≈ 1500 tokens
- 30章 = 45000 tokens（占用Claude 200K的22%）
- 100章 = 15万 tokens（占用75%，挤掉其他内容）
- 必须分层压缩：原始文本→章节摘要→卷摘要

**问题2：综合提示词混了"永恒"和"会变"的**
- 教程原则 = 永恒不变（L0层）
- 整体大纲 = 写作过程中半稳定（写完一卷才更新）（L1层）
- 早期细致大纲 = 高频变化（每写几章就要推进）（L1层）
- 基础设定 = 永恒不变（L0层）+ 详细设定按需调用（L3层）
- 混在一起会导致AI分不清什么能改、什么不能改

**问题3：缺少"当前任务"层**
- AI需要明确的"本章写作任务"指令
- 没有任务层，AI会写得很泛、不聚焦
- 这是每章必填、最影响输出质量的层

## 第三部分：5层模板（可直接复制使用）

### L0 永恒核心模板（~2500 tokens，一次制作永不变）

```markdown
# 番茄小说AI写作 - 永恒核心规则

## 1. 写作心法（不可违背）
- 稳定更新是底线：断更=自杀，每章3000字，日更6000字（2章）
- 节奏管理：5-10章一个小爽点，30-40章一个大爽点，100章一个超级爽点
- 钩子九连环：每章必有钩子，章末必留新钩子，一个钩子引下一个
- 人物是灵魂：设定是外壳，情节是骨架，人物关系转变是引擎
- 合理中的荒诞：在结构自洽前提下，创造让人眼前一亮的转折
- 爆得快更要热得长：开局有冲击，中段不松懈，震荡上升而非一路高歌

## 2. 开篇禁忌（前30章严格遵守）
- ❌ 千万不要第一时间抛出大量世界观、体系设定
- ❌ 前五章内不要出现任何没有意义的人
- ❌ 前三章不要夹杂多条支线
- ✅ 剧情切入（推荐新手）：对话、危机、金手指降临
- ✅ 前200字必须抛出爆炸性设定（番茄都市脑洞刚需）

## 3. 钩子设计5种类型
- 剧情反转类：学会"拐弯"，山路十八弯
- 情绪向钩子：天崩开局，共情拉满
- 信息差钩子：用悬疑式信息差
- 反差/锚点钩子：现实与幻想的反差
- 竞争/对比钩子：主角与反派目标一致手段不同

## 4. 人物塑造规则
- 1个核心性格标签 + 2-3个反差标签（教程6.3+6.5）
- 行为底层逻辑：通篇遵循，违背即崩塌（教程6.4）
- 重立场和选择，轻善恶：反派要有自洽逻辑（教程6.4方法3）
- CP作为整体设计：CP类型+关系+天然冲突+致命吸引+糖点（教程6.2）
- 标签互补：男女主性格冲突，糖点是"例外和偏爱"

## 5. 章节结构硬性规则
- 每章3000字（番茄主流2000-3000字）
- 开头200-300字：延续上章钩子+快速进入场景
- 主体2400-2500字：本章事件+至少1个冲突/爽点+推动1条线
- 结尾200-300字：留新钩子+引向下章
- 每章至少1个对话场景、1个细节描写
- APP提示每章至少1次（除纯对话章）

## 6. 章节标题规则
- 6-10字为主（80%），11-15字次之（20%）
- 短词组为主（如"反向设局"），陈述句次之
- 不带书名号/引号/感叹号
- 每章标题必须有钩子

## 7. 代入感6技巧
- 第一人称内心独白（口语化、有情绪）
- 细节真实化（用具体可感细节代替抽象描述）
- 五感描写（每段1-2个感官，不全写）
- 情绪传递链（通过动作表情对话传递情绪）
- 场景代入（描写读者熟悉的场景）
- 信息差共鸣（让读者知道主角不知道的信息）

## 8. 本书专属规则（不可违背）

### 主角林舟
- 核心标签：嘴贱心软
- 行为底层逻辑：用最大恶意揣测危机，用最大善意对待病人
- 不能做：不杀人、不害无辜、不背叛病人
- 必须做：见到危机不能不管、收挂号费20元、凌晨三点研究病例
- 口头禅："挂号费20，记得交"

### APP规则
- 危机等级5级：低危/中危/高危/致命/临界（不出现1级/S级）
- 危机类型7类：D疾病/P心理/F财务/R人际/S安全/A学业/H家庭
- 提示格式3种：常规【D-姓名：描述，概率%，可干预窗口】/红色预警【⚠️红色预警-姓名：危机类型，等级LvX，位置，倒计时】/解除反馈【危机解除-姓名：进度+XX%，获得善意值XXX】
- 善意值命名固定：善意值（不出现功德值/救世点）

### 禁忌
- 不出现"文抄公"（主角不抄地球作品）
- 不出现纯善纯恶（反派要有自洽逻辑）
- 不出现系统签到/任务奖励（金手指靠解决危机获得善意值）
- 不出现后宫流水账（多女主错峰，修罗场错峰）
```

## 第五部分：制作工作流

### Phase 1：开新书前（一次性制作，约2小时）

1. **制作L0永恒核心**
   - 从教程提取核心原则（已在本文件第三部分提供）
   - 加入本书专属规则（主角/APP/禁忌）
   - 保存为 `L0_永恒核心.md`

2. **制作L1故事骨架（卷一版）**
   - 从v2大纲压缩4卷概要
   - 填入卷一的事件弧、爽点、角色、伏笔、升华点
   - 保存为 `L1_故事骨架_卷一.md`

3. **制作L3设定索引**
   - 从P0P1设定集建立索引表
   - 按角色/场景/伏笔/能力分类
   - 保存为 `L3_设定索引.md`

4. **制作L2/L4模板**
   - 复制本文件第三部分的模板
   - 保存为 `L2_近章上下文_模板.md` 和 `L4_当前任务_模板.md`

### Phase 2：每章写作前（约10分钟）

1. **更新L2近章上下文**
   - 填入最近3章摘要（AI辅助生成）
   - 粘贴上一章末尾500字原文
   - 填入本章详细章纲（从v2大纲复制）
   - 填入下一章钩子预告

2. **检索L3按需设定**
   - 根据本章章纲，识别出场角色
   - 从P0P1设定集复制相关角色设定
   - 识别APP使用，复制能力设定
   - 识别场景，复制物理描述
   - 识别伏笔，复制状态

3. **填写L4当前任务**
   - 从章纲转化核心任务
   - 列必含元素
   - 指定风格和禁忌

4. **组装提示词**
   - 按L0→L1→L2→L3→L4顺序粘贴
   - 发送给AI

### Phase 3：每章写作后（约5分钟）

1. **生成章节摘要**
   - 让AI用200字总结本章
   - 保存到 `章节摘要/第X章.md`

2. **更新进度追踪**
   - 更新L2的"当前进度"部分
   - 更新善意值累计
   - 更新APP等级

3. **每10章自检**
   - 对照P0P1设定集末尾的自检表
   - 检查设定一致性
   - 如有冲突，修正

### Phase 4：每卷结束（约30分钟）

1. **更新L1故事骨架**
   - 切换到下一卷的事件弧
   - 更新角色清单（如有新角色）
   - 更新反派清单（如倒戈/死亡）
   - 更新伏笔状态

2. **生成卷摘要**
   - 让AI用1000字总结本卷
   - 保存到 `卷摘要/卷X.md`

3. **更新L0（如有需要）**
   - 如发现新的禁忌规则，加入L0
   - 如主角行为底层逻辑有微调，更新L0

## 第七部分：实操建议

### 1. 文件组织结构

```
/home/z/my-project/novel/
├── prompts/
│   ├── L0_永恒核心.md          (一次制作)
│   ├── L1_故事骨架_卷一.md     (每卷更新)
│   ├── L1_故事骨架_卷二.md
│   ├── L2_近章上下文_模板.md   (每章复制填写)
│   ├── L3_设定索引.md          (一次制作)
│   ├── L4_当前任务_模板.md     (每章复制填写)
│   └── 组装提示词_模板.md      (空白模板)
├── chapters/
│   ├── 第001章_入职第一天的奇怪APP.md
│   ├── 第002章_天台上的第一单.md
│   └── ...
├── summaries/
│   ├── 章节摘要/
│   │   ├── 第001章摘要.md
│   │   └── ...
│   └── 卷摘要/
│       ├── 卷一摘要.md
│       └── ...
├── settings/
│   ├── P0P1详细设定集.md       (完整设定)
│   └── 自检表.md
└── progress/
    ├── 进度追踪.md             (每章更新)
    └── 善意值账本.md           (每章对账)
```

### 2. 每章写作流程（10步法）

1. 打开 `L2_近章上下文_模板.md`，另存为 `L2_第X章.md`
2. 生成上一章摘要（AI辅助），填入L2
3. 粘贴上一章末尾500字原文到L2
4. 从v2大纲复制本章章纲到L2
5. 从v2大纲复制下一章钩子预告到L2
6. 打开 `L4_当前任务_模板.md`，另存为 `L4_第X章.md`，填写任务
7. 查 `L3_设定索引.md`，识别本章需要的设定，从P0P1设定集复制到 `L3_第X章.md`
8. 组装：L0 + L1当前卷 + L2_第X章 + L3_第X章 + L4_第X章
9. 发送给AI，生成章节正文
10. 保存章节到 `chapters/`，生成摘要到 `summaries/`，更新进度

### 3. 质量保障

- **每章自检**：写完后对照L4的"必含元素"清单，缺则补
- **每10章自检**：对照P0P1设定集的自检表（15项）
- **每卷自检**：对照L1的卷目标，检查是否完成
- **冲突应急**：发现设定冲突，立即停笔，回查设定集修正

### 4. 优化建议

- **L0不要频繁改动**：L0是核心，改一次影响全书。如必须改，记录改动原因和影响范围
- **L2摘要是关键**：好的摘要能让AI保持长期一致性。建议用"事件+冲突+钩子+推动线"四要素格式
- **L3按需调用**：不要贪多，每章只检索本章真正需要的设定。过多会干扰AI
- **L4越具体越好**：AI的输出质量与L4的具体程度正相关。"写第5章"不如"写第5章，3000字，张老师在天台，林舟用红烧肉破冰，章末留苏晚债务钩子"

## 第八部分：与你的3层方案对比

| 维度 | 你的3层方案 | 5层洋葱模型 |
|-----|-----------|-----------|
| 层数 | 3层 | 5层 |
| 区分永恒vs会变 | ❌ 混在综合提示词 | ✅ L0永恒/L1半稳定/L2每章变 |
| 已有内容处理 | 全塞（30章后超上下文） | 摘要+原文片段（L2，永远4000 tokens） |
| 设定调用 | 笼统说"相关部分" | 索引表精确检索（L3） |
| 当前任务 | ❌ 缺失 | ✅ L4专门层 |
| token预算 | 不可控 | 12000 tokens固定 |
| 制作工作量 | 一次制作 | L0/L1/L3一次制作，L2/L4每章填 |
| 适配长篇 | 100章后崩 | 500章稳定 |
| 可维护性 | 设定改动影响大 | 分层改动，影响可控 |

**结论**：你的3层方案方向正确，5层洋葱模型是在其基础上的工程化优化，核心改进是"分层压缩"+"索引检索"+"任务层"。

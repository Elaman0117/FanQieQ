# 世界书框架（词条分类体系 + 命名规则 + 模板）

> **使用方式**：本文件定义世界书的结构与维护规则。每章写完后，AI按此框架更新词条；每章开写前，AI按此框架检索相关词条。
> **核心理念**：用"词条化+AI自动检索"替代"人工预判注入"，实现可扩展、自更新、防遗忘的设定管理。

---

## 一、世界书整体架构

### 1.1 三层分类体系

```
世界书
├── 第一层：永久词条（P0级，全书不变）
│   ├── 角色档案（5女主+主角+核心配角）
│   ├── 反派档案（4层金字塔）
│   ├── APP规则（机制+格式+限制）
│   ├── 善意值体系（获取+兑换+价格表）
│   ├── 地理设定（南江市+南江大学+校医室）
│   └── 伏笔总表（7条长线伏笔）
│
├── 第二层：动态词条（P1级，随剧情更新）
│   ├── 章节事件库（每章写完新增一条）
│   ├── 善意值对账表（每章更新）
│   ├── 女主感情进度表（每章更新）
│   ├── 角色状态表（位置/关系/已知信息）
│   ├── 已兑换能力清单（每章更新）
│   └── 伏笔进度表（埋设/推进/回收状态）
│
└── 第三层：临时词条（P2级，事件弧结束后归档）
    ├── 当前事件弧设定（如"李明脑瘤案"专属设定）
    ├── 章节级特殊设定（如Ch33"APP失灵"的临时规则）
    └── 已回收伏笔档案（回收后移入此层）
```

### 1.2 词条命名规则

**命名格式**：`[分类前缀]_[子类]_[具体名]`

**分类前缀**（固定）：
| 前缀 | 含义 | 示例 |
|-----|------|------|
| `CHAR` | 角色 | CHAR_HEROINE_苏晚 |
| `VILAIN` | 反派 | VILAIN_MID_赵天宇 |
| `APP` | APP规则 | APP_RULE_危机等级 |
| `POINT` | 善意值 | POINT_PRICE_医术类 |
| `GEO` | 地理 | GEO_BUILD_校医室 |
| `PLOT` | 伏笔 | PLOT_F1_APP来源 |
| `CHAP` | 章节事件 | CHAP_007_反向设局 |
| `ARC` | 事件弧 | ARC_01_李明脑瘤案 |
| `STAT` | 状态 | STAT_苏晚_感情进度 |
| `TEMP` | 临时 | TEMP_ARC01_李明家庭 |

## 六、AI更新规则（自主判断模式）

### 6.1 核心原则：AI自主判断

AI写完一章后，**对每条潜在更新拥有完全的自主判断权**，可以根据本章实际内容决定：
- 是否更新某个已有词条
- 是否新建一个词条
- 是否跳过本次更新

**判断标准**：只有当本章内容确实产生了"会影响后续章节写作的信息"时，才需要更新/新建。如果本章只是过场、对话、铺垫，没有产生新的设定信息，AI可以零更新。

### 6.2 候选更新清单（AI按需勾选）

AI写完一章后，逐项检查以下候选更新，**自主决定**是否执行：

| 候选项 | 触发条件 | AI决策 |
|-------|---------|-------|
| 新建`CHAP_[本章]_事件` | 每章都应创建（章节摘要） | 通常执行，除非章节无实质内容 |
| 更新`STAT_林舟_已兑换能力` | 本章有新兑换/能力升级 | 如有则更新 |
| 更新`POINT_PRICE_前50章兑换表` | 本章有善意值变动 | 如有则更新 |
| 更新`STAT_[角色]_当前状态` | 角色位置/情绪/已知信息有变化 | 如有显著变化则更新 |
| 更新`STAT_5女主感情进度总表` | 女主感情有进展 | 如有进展则更新 |
| 新建`CHAR_/VILAIN_`词条 | 新角色出场且有戏份 | 如角色会再出场则新建 |
| 新建`GEO_BUILD_`词条 | 新场景出场 | 如场景会再出现则新建 |
| 更新`PLOT_FX_`词条 | 伏笔有推进/回收 | 如有则更新 |
| 新建`PLOT_FX_`词条 | 发现新伏笔 | 如判断为长线伏笔则新建 |
| 更新APP功能解锁状态 | APP升级 | 如有则更新 |
| 更新`ARC_XX_`状态 | 事件弧结束 | 如弧结束则归档 |
| 新建`TEMP_`词条 | 临时规则/事件弧专属设定 | 如需要则新建 |

### 6.3 AI更新决策流程

每章写完后，AI按以下流程决策：

```
Step 1：扫描本章内容
  - 本章出现了哪些角色？
  - 本章出现了哪些场景？
  - 本章是否有APP兑换/升级？
  - 本章是否有善意值变动？
  - 本章是否有伏笔埋设/推进/回收？
  - 本章是否有女主感情进展？
  - 本章是否有新角色/新场景/新设定？

Step 2：对照候选清单（6.2）
  - 逐项判断"是否触发条件"
  - 对触发的项，进一步判断"是否真的需要记录"
    - 该信息后续会用到吗？
    - 该信息是否会与已写章节冲突？
    - 该信息是否已被其他词条覆盖？

Step 3：执行更新（自主决定）
  - 对判断为"需要更新"的项，执行更新
  - 对判断为"不需要更新"的项，跳过
  - 对判断为"需要新建"的项，按模板创建

Step 4：生成更新报告
  - 列出本次更新的词条
  - 列出本次跳过的项及原因
  - 列出本次新建的词条
```

## 七、世界书维护原则

### 7.1 一致性原则
- 永久词条（P0级）全书不变，如需修改必须回查所有已写章节
- 动态词条（P1级）每章更新，但更新必须与永久词条一致
- 临时词条（P2级）事件弧结束后归档，不再修改

### 7.2 防漂移原则
- 善意值数字每章对账，避免漂移
- 女主感情进度严格按预设节奏，不超前
- 反派行为必须符合其行为底层逻辑表
- APP格式必须按三种固定格式

### 7.3 可扩展原则
- 新角色出场自动新建词条
- 新场景自动新建地理词条
- 新伏笔自动新建伏笔词条
- 写到200章时不需要人工维护，AI自动检索+更新

### 7.4 防遗忘原则
- 所有伏笔永久保存在PLOT词条中
- 已回收伏笔移入"已回收伏笔档案"，不删除
- 远距离伏笔通过词条检索自动激活

## 🚀 快速开始

### 第一次使用

1. **阅读 4 个核心文件**（按顺序）：
   - `01_综合系统提示词.md`（最重要，通读3遍）
   - `02_世界书框架.md`（通读2遍）
   - `03_章节任务指令模板.md`（通读1遍）
   - `04_写作工作流SOP.md`（通读1遍）

2. **浏览世界书词条**（33个，已预填好初始内容）

3. **开始写第1章**：
   - 把 `01_综合系统提示词.md` 全文作为系统提示词
   - 复制 `03_章节任务指令模板.md` 的标准模板
   - 参考 `01` 文件第十三层 Ch1 部分填写本章任务
   - 发给AI生成

### 日常写作流程

**支持agent的AI**：参考 `04_写作工作流SOP.md`，AI自动检索+生成+更新。

**不支持agent的AI（手动）**：参考 `05_手动工作流指南.md`，每章2-3轮对话：

```
第1轮：喂上下文（01系统提示词+上一章正文+章节任务指令+相关世界书词条）→ AI生成章节正文
第2轮：请求世界书更新 → AI输出更新内容 → 手动粘贴到文件
```

手动工作流详细操作见 `05_手动工作流指南.md`。

## 📋 核心设计

### 三段式上下文架构

```
┌─────────────────────────────────┐
│ ① 01_综合系统提示词（永久）      │ ← 每次放最开头
│   教程原则+整体大纲+早期大纲+设定 │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│ ② 已写正文（最近原文+早期摘要）  │ ← 后续按需处理
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│ ③ 章节任务指令（用户填写）       │ ← 每章填一次
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│ ④ 世界书词条（AI自动检索注入）   │ ← AI根据③自动检索
└─────────────────────────────────┘
              ↓
         AI生成章节正文
              ↓
┌─────────────────────────────────┐
│ ⑤ 写完后：AI自主更新/新建词条    │ ← AI按02文件规则
└─────────────────────────────────┘
```

### AI自主判断机制

**世界书更新**：AI对每条潜在更新拥有完全自主判断权
- 是否更新某个已有词条 → AI决定
- 是否新建一个词条 → AI决定
- 是否跳过本次更新 → AI决定

**判断标准**：只有当本章内容产生了"会影响后续章节写作的信息"时，才更新/新建。过场/对话/铺垫章节可零更新。

详见 `02_世界书框架.md` 第6节。

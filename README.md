**不支持agent的AI（手动）**：参考 `05_手动工作流指南.md`，每章2-3轮对话。

## 🔄 与二次流程的协作

### 整体架构

```
一次流程（本文件夹）
  ├─ 模型A → 生成第n章候选
  ├─ 模型B → 生成第n章候选
  ├─ 模型C → 生成第n章候选
  └─ 最强模型 → 生成第n章候选
         ↓
  用户只复制各模型的"第n章正文"到 二次流程/候选章节/
         ↓
二次流程
  ├─ 读取候选章节 + 前1~n-1章最终正文 + 世界书
  ├─ 融合生成最终第n章
  └─ 更新世界书
         ↓
  用户手动同步回一次流程：
  ├─ 复制最终第n章到 一次流程/正文/
  └─ 复制更新的世界书词条到 一次流程/世界书/
         ↓
  一次流程开始第n+1章循环
```

### 同步操作

**从一次流程 → 二次流程**（每章生成后）：
- 复制各模型生成的第n章正文到 `二次流程/候选章节/`
- 命名：`模型A_第n章.md`、`模型B_第n章.md` ...

**从二次流程 → 一次流程**（融合完成后）：
- 复制 `二次流程/最终输出/第XXX章_标题.md` 到 `一次流程/正文/`
- 复制二次流程更新的世界书词条到 `一次流程/世界书/` 对应子文件夹

## 📋 核心设计

### 三段式上下文架构

```
┌─────────────────────────────────┐
│ ① 01_综合系统提示词（永久）      │ ← 每次放最开头
│   教程原则+整体大纲+早期大纲+设定 │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│ ② 已写正文（最近原文+早期摘要）  │ ← 后续按需处理
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│ ③ 章节任务指令（用户填写）       │ ← 每章填一次
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│ ④ 世界书词条（AI自动检索注入）   │ ← AI根据③自动检索
└─────────────────────────────────┘
              ↓
         AI生成章节正文
              ↓
┌─────────────────────────────────┐
│ ⑤ 写完后：AI自主更新/新建词条    │ ← AI按02文件规则
└─────────────────────────────────┘
```

### AI自主判断机制

**世界书更新**：AI对每条潜在更新拥有完全自主判断权
- 是否更新某个已有词条 → AI决定
- 是否新建一个词条 → AI决定
- 是否跳过本次更新 → AI决定

**判断标准**：只有当本章内容产生了"会影响后续章节写作的信息"时，才更新/新建。

详见 `02_世界书框架.md` 第6节。
